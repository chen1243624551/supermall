### 选项卡的数据非常庞大,而且客户往下拉还会继续加在数据,所以需要用一个变量来保存数据.
### 这个案例中整体有三部分数据,一个是点击 流行 时候的数据,点击 新款 的时候数据和点击 精选 的时候数据
### 先一次性把每一部分的第一页的数据线都请求过来,分成三部分保存在同一个变量中
  goods:{
    //page表示当前是第几页的数据,list表示当前已经加载的数据个数,随着用户下拉,页数随着改变,数据也增加
  'pop':{page:1,list:[30]},
  'new':{page:1,list:[30]},
  'sell':{page:1,list[30]}
}
